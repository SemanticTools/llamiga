# LLaMiga V0.9 — Error Taxonomy & Configurable Retry

## Goals

1. Make `complete()` return **consistent, distinguishable errors** across all six provider plugins.
2. Make retry behavior **configurable** at library, plugin, session, and (plugin, model) scopes.
3. Kill the "while-loop falls off without returning" bug class by construction.

## Out of scope

- Per-call retry override (callers configure via session).
- Replacing the existing `overrideConfig` plumbing at `src/index.mjs:654` (used by chain mode).
- Changing the success-path shape returned by `complete()`.

---

## 1. Error taxonomy

Six categories. Errors are thrown as plain `Error` instances with the following fields:

```js
err.code        // one of the six codes below
err.status      // HTTP status when applicable, else null
err.provider    // 'Anthropic' | 'OpenAI' | 'Gemini' | 'Grok' | 'Mistral' | 'Ollama'
err.message     // human-readable string (provider + model prefix recommended)
err.raw         // upstream response data (HTTP errors only)
  .headers      //   response headers as a plain object — retry-after, x-request-id, rate-limit headers
  .responseBody //   parsed JSON body, or raw text if JSON parse failed
err.cause       // underlying Error (standard ES2022 attribute) — fetch throw (NETWORK), JSON parse failure, etc.
```

`.raw` groups data that came back from the HTTP response (body + headers). `.cause` is set whenever there's an underlying thrown exception — using the standard ES2022 attribute means debuggers, log formatters, and code that walks `.cause` chains pick it up automatically. Fields that don't apply for a given error (e.g. no `.raw` on a NETWORK error since no response arrived) are absent rather than `null`.

| Code | Trigger | Retry-worthy |
|---|---|---|
| `RATE_LIMIT` | 429 transient (provider-side throttling) | yes |
| `QUOTA_EXHAUSTED` | account credits / token budget exhausted | **no** |
| `AUTH` | 401, 403 — bad/missing/expired key, key lacks permission | no |
| `CLIENT` | other 4xx — bad request, unknown model, payload too large, context overflow | no |
| `SERVER` | 5xx — provider down or overloaded | yes |
| `NETWORK` | `fetch` threw / no HTTP response (DNS, TCP reset, timeout) | yes |

### Per-provider classification rules

Lives in `src/llm/common/errors.mjs`:

```js
classifyHttpError(response, body, provider) -> Error
classifyNetworkError(err, provider)         -> Error
```

| Provider | How to distinguish |
|---|---|
| **OpenAI / Grok / Mistral** | Read `body.error.type` and `body.error.code`. `insufficient_quota` → `QUOTA_EXHAUSTED`. `invalid_api_key` → `AUTH`. `rate_limit_exceeded` → `RATE_LIMIT`. `model_not_found` → `CLIENT`. |
| **Anthropic** | Read `body.error.type`. `authentication_error` / `permission_error` → `AUTH`. `rate_limit_error` → `RATE_LIMIT`. `overloaded_error` → `SERVER`. `invalid_request_error` → `CLIENT`, **unless** message contains "credit balance" → `QUOTA_EXHAUSTED`. |
| **Gemini** | Read `body.error.status`. `RESOURCE_EXHAUSTED` is overloaded — disambiguate via `body.error.details[].reason`: `RATE_LIMIT_EXCEEDED` → `RATE_LIMIT`, `QUOTA_EXCEEDED` → `QUOTA_EXHAUSTED`. `UNAUTHENTICATED` / `PERMISSION_DENIED` → `AUTH`. `INVALID_ARGUMENT` → `CLIENT`. |
| **Ollama** | Local. In practice only `SERVER` (5xx) and `CLIENT` (4xx). No `AUTH` / `QUOTA` / `RATE_LIMIT` paths. |

`classifyNetworkError` covers `fetch` throws (no response object): unconditional `NETWORK`.

### Error message format

`[<Provider>/<model>] <category-specific reason>` — e.g. `[Anthropic/claude-3-opus] Rate limit exceeded`. Helps logs.

## 2. Configurable retry

### Library defaults (`src/llm/common/retry.mjs`)

```js
{
  maxAttempts: 3,
  backoff: 'exponential',                          // 'fixed' | 'exponential'
  baseMs: 1000,
  maxMs: 30000,
  jitter: true,                                    // ±25% randomization
  honorRetryAfter: true,                           // header overrides computed delay, capped at maxMs
  retryOn: ['RATE_LIMIT', 'SERVER', 'NETWORK'],
  totalTimeoutMs: null,                            // optional hard ceiling across all attempts
  onRetry: null,                                   // ({attempt, error, delayMs}) => {} — observability hook
}
```

### The `withRetry` helper

```js
async function withRetry(fn, opts) { ... }
```

Owns the retry loop. Guarantees:
- Always exits via `return` or `throw` — no falling off the end.
- On exhaustion, throws the **last classified error** (e.g. `RATE_LIMIT`), not a generic "exceeded" string.
- Increments attempt counter on every iteration (fixes the catch-block-without-increment latent bug).
- Honors `totalTimeoutMs` as a hard ceiling — if the next computed delay would exceed remaining budget, throws immediately.
- Calls `onRetry({attempt, error, delayMs})` before each sleep.

### Plugin-level defaults

Each plugin module exports a `defaultRetry` object alongside its existing exports:

```js
export const defaultRetry = { /* overrides on top of library defaults */ };
```

- Most plugins: empty object (use library defaults).
- Ollama: `{ baseMs: 20000 }` — replaces the current hardcoded 20s.

### Merge chain

```
library defaults                               (common/retry.mjs)
  ↓ shallow merge
plugin.defaultRetry                            (per plugin module)
  ↓ shallow merge
session-wide config (setConfig with 1 arg)     (sessionConfig slot)
  ↓ shallow merge
plugin-wide config (setConfig with 2 args)     (pluginConfigs['plugin::default'])
  ↓ shallow merge
plugin+model config (setConfig with 3 args)    (pluginConfigs['plugin::model'])
```

Last writer wins. Merge is shallow over the `retry` block (each user-supplied `retry` object is `{...defaults, ...userRetry}` — they don't need to supply every field).

`retry: false` at any layer disables retry entirely from that layer down.

## 3. Session API changes (`src/index.mjs`)

### `setConfig` — arity polymorphism

Discriminate by `typeof p1 === 'string'`.

```js
session.setConfig({retry, ...})                        // 1 arg  → session-wide
session.setConfig('OpenAI', {retry, ...})              // 2 args → plugin, all models (existing)
session.setConfig('OpenAI', 'gpt-4o', {retry, ...})    // 3 args → plugin + model (existing)
```

- 1-arg form stores into a new `sessionConfig` slot on the session (not in `pluginConfigs`).
- All three forms inject `framework` and `session: this` into the stored object (uniform shape for plugins).
- 1-arg form skips the "No plugin found" validation (no plugin name to validate).

### `getConfig` — same arity pattern

```js
session.getConfig()                       // returns sessionConfig
session.getConfig('OpenAI')               // existing — falls back through sessionConfig if absent
session.getConfig('OpenAI', 'gpt-4o')     // existing — falls back through plugin → sessionConfig
```

Lookup chain inside `getConfig(plugin, model)`:

```
pluginConfigs['plugin::model']
  → pluginConfigs['plugin::default']
  → sessionConfig
  → undefined
```

### `clearConfig` — same arity pattern

```js
session.clearConfig()                     // clear sessionConfig
session.clearConfig('OpenAI')             // existing
session.clearConfig('OpenAI', 'gpt-4o')   // existing
```

### Config merge point

Inside `session.ask()` (around `index.mjs:662`), before calling `plugin.complete()`, the session assembles the **merged** config to pass in. This is the single place that walks the merge chain. The plugin receives one fully-merged config and never re-merges.

## 4. Plugin changes (all six)

Each plugin's `complete()` collapses from a hand-rolled `while (retries < maxTries)` to:

```js
return withRetry(
  async () => {
    // 1. transform messages (unchanged)
    // 2. fetch
    // 3. if (!response.ok) throw classifyHttpError(response, body, providerName)
    // 4. parse + return success object
  },
  config.retry
);
```

`fetch` throws are caught by `withRetry` and run through `classifyNetworkError`.

Each plugin:
- Adds `export const defaultRetry = {...}` (Ollama gets `{baseMs: 20000}`; the others get `{}`).
- Removes the hand-rolled retry loop, the `throw new Error('Max retries exceeded')` added earlier in this branch, and the catch-block `continue` branch.
- Replaces ad-hoc `throw new Error(...)` with `classifyHttpError(...)`.
- `pgOpenAI.mjs` does the same in both `complete()` and `completeStreaming()`.

Other behavior (message transformation, role translation, response parsing, success-path shape) is unchanged.

## 6. Behavior changes visible to callers

1. **`complete()` throws categorized errors** with `.code`. Callers reading `err.message` keep working; callers using `instanceof Error` keep working; new code can switch on `err.code`.
2. **`AUTH` / `QUOTA_EXHAUSTED` / `CLIENT` fail fast** instead of burning 3 retries. Calls that used to take ~9s to fail now fail in ~100ms. Intended improvement, but visible.
3. **Exhaustion throws the last classified error**, not `'Max retries exceeded'`. A 429 storm surfaces as `RATE_LIMIT`, useful for caller logic.
4. **`Retry-After` headers honored** (capped at `maxMs`) — replaces fixed 3s sleep.
5. **No per-call retry override** — `config.retry` originating from a per-call argument is no longer honored. Use `setConfig` instead.
6. **The patched `node_modules/.../pgAnthropicAI.mjs`** (the original incident) no longer needs patching once a release goes out — the bug is fixed structurally for all six plugins.
